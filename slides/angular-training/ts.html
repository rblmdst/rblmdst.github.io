Introduction

What is TypeScript ?

TypeScript is an open-source programming language from Microsoft.

TypeScript = JavaScript + Types

TypeScript is compiled to Javascript.

Why TypeScript ?

1- TypeScript adds type to Javascript , that make the code easy to read and maintain.

Javascript 


              // test.js
              const add = (x, y) => x + y;
              add(1, 3) // 4
              add(1, '3') // '13'

Typescript 


              // test.ts
              const add = (x: number, y: number) => x + y;
              add(1, 3) // 4
              add(1, '3') // compiler error before JS is  even  produced: '3' is not a number

Why TypeScript ?

2- Allow to use ES6 features in old browser (ES5) it needs to be transpiled to ES5.

( ES6 is not supported by all browsers.)

some important points to keep in mind

TypeScript :

  • Is a typed superset of JavaScript and compiles to plain JavaScript.
  • Files have the extension " .ts".
  • Is not intended to replace JavaScript in any way. It adheres to all principles of JavaScript and just adds types on top of it.
  • Looks very familiar to any other typed OOPs languages like C# and Java.
  • Is not executed on the browser. Instead, the transpiled code gets executed in the browser.

TypeScript Compiler (TSC)

Installation 


            # install the typescript compiler globally
            $ npm install -g typescript

Check the version of TypeScript installed 


            $ tsc -v
            Version 2.3.3

Compilation

Compilation is done by using the tsc command as follow :


            tsc [options] [file ...]

Examples:


            $ tsc hello.ts
            $ tsc ./utils/math.ts index.ts
            $ tsc --outFile file.js file.ts

Compilation using a configuration file

By default the compiler checks the existence of a tsconfig.json file in the current directory.

if it exist he used it. If not it use the default compiler options.

Compilation using a configuration file

The configuration file contains information that tells the compiler :

  • how to make the compilation;
  • where to compile the file to;
  • etc.

Create a configuration file

A tsconfig.json file (if no one already exist) can be generated by using :


            $ tsc --init
            message TS6071: Successfully created a tsconfig.json file.

Example of tsconfig.json file of a basic Angular project : 


            {
              "compilerOptions": {
                "module": "es2015",
                "outDir": "./dist/out-tsc",
                "target": "es5",
                "emitDecoratorMetadata": true,
                "experimentalDecorators": true,
                "noImplicitAny": true,
                "removeComments": false,
                "sourceMap": true
            },
            "exclude": [
              "node_modules",
              "dist/"
            ]
          }

Custom configuration file

A custom configuration file can be passed to tsc by using the -p or --project option


            $ tsc --project tsconfig.prod.json

Watch Mode

Instead of hiting tsc every time a file changed in your directory

tsc allows to make the compilation automatically on file(s) change.


              $ tsc -w [file ...]

Types

Primitive Types

TypeScript inherits five primitive types from JavaScript

  • string;
  • number;
  • boolean;
  • undefined;
  • null.

Primitive Types

string : can be assigned with single, double quotes or backtick.

number : stores any number as integer or float.

boolean : stores true or false values.

undefined : refers to a variable that has no value or object assigned to it.

null : refers to a variable that has been assigned the value null.

Primitive Types 


              let isDone: boolean = false;
              let height: number = 6;
              let name: string = "bob";
              let x; // undefined
              let nullable = null;

Non-Primitive Types

Apart from the primitive types, we also have :

  • array;
  • tuple;
  • enum;
  • any;
  • void.

Array

Array allows you to work with arrays of values.

Array types can be written in one of two ways.


              let list: number[] = [1, 2, 3];
              let list: Array<number> = [1, 2, 3];

Tuple

Tuple allows you to express an array where the type of a fixed number of elements is known, but need not be the same.

Eg: representing a value as a pair of a string and a number.


              // Declare a tuple type
              let x: [string, number];
              // Initialize it
              x = ["hello", 10]; // OK
              // Initialize it incorrectly
              x = [10, "hello"]; // Error

Tuple

When accessing an element with a known index, the correct type is retrieved : 


            // Declare a tuple type
            let x: [string, number];
            // Initialize it
            x = ["hello", 10]; // OK

            console.log(x[0].substr(1)); // OK
            console.log(x[1].substr(1)); // Error, 'number' does not have 'substr'

Tuple

When accessing an element outside the set of known indices, a union type is used instead : 


              // Declare a tuple type
              let x: [string, number];
              // Initialize it
              x = ["hello", 10]; // OK

              x[3] = "world"; // OK, 'string' can be assigned to 'string | number'
              console.log(x[5].toString()); // OK, 'string' and 'number' both have 'toString'
              x[6] = true; // Error, 'boolean' isn't 'string | number'

Any

Any is a kind of supertype of all types.

Any is used whenever TypeScript is unable to infer a type. 


            let notSure: any = 4;
            notSure = "maybe a string instead";
            notSure = false; // okay, definitely a boolean

Enum

Enum is a way of giving more friendly names to sets of values. 


            enum Color {Red, Green, Blue};
            let c: Color = Color.Green;

Start at 1 instead of 0 :


            enum Color {Red = 1, Green, Blue}
            let c: Color = Color.Green;

Manually set all the values in the enum :


            enum Color {Red = 1, Green = 2, Blue = 4}
            let c: Color = Color.Green;

Void

Void is a little like the opposite of any : the absence of having any type at all.

Void is commonly used as the return type of functions that do not return a value. 


            function showMessage(data: string): void {
              alert(data);
            }
            showMessage('hello');

Declaring variables of type void is not useful because you can only assign undefined or null to them :


            let unusable: void = undefined;

Object

Object is a type that represents the non-primitive type, i.e. any thing that is not number, string, boolean, symbol, null, or undefined. 


              declare function create(o: object | null): void;

              create({ prop: 0 }); // OK
              create(null); // OK

              create(42); // Error
              create("string"); // Error
              create(false); // Error
              create(undefined); // Error

Interfaces

Interfaces are abstract descriptions of things, and can be used to represent any non-primitive JavaScript object.

They produced no code (ES6 or ES5), and exist only to describe types to tsc.

They are compile-time constructs that are used to define the contract for a class by just declaring the methods and fields.


            // declare the interface
            interface Greet {
              greetUser(message: string): void;
            }

            // To implement the interface, use implements
            class Greeting implements Greet {
              // The class implementing the interface should  defines all members of the interface
              greetUser(msg: string) {
                console.log(msg);
              };
            }

They can be applied to functions, properties, and also to arrays.

Example of an interface describing a function :


              interface Callback {
                (error: Error, data: any): void;
              }
              function callServer(callback: Callback) {
                  callback(null, 'hi');
              }
              callServer((error, data) => console.log(data)); // 'hi'
              callServer('hi'); //tsc error

Example of an interface describing an object literal : 


              interface Action  {
                type: string;
              }

              let a: Action = {
                type: 'literal'
              }

Functions

All that we have learned in the ES6 part about functions are valid in typescript.


              // Named function
              function log(msg: string) {
                console.log(msg);
              }
              // anonymous function
              let double = function (x: number) { return 2 * x};
  
              /* default parameter */
              function hello(name: string = 'World') {
                console.log(`Hello ${name}`);
              }
  
              hello(); // Hello World
              hello('Alex'); // Hello Alex

Optional parameters


              function hello(name?: string) {
                if (name) {
                  console.log(`Hello ${name}`)
                }
              }
  
              hello(); // nothing is displayed
              hello('Alex'); // Hello Alex

Classes

Declaration


                class Person {
                  /* attributes/properties can be public or private */
                  private _name: string;
  
                  constructor(name: string) {
                    this._name = name;
                  }
  
                  /* methods can also can be public or private */
                  public introducedMe() {
                    console.log(`Hello, I am ${this.name} !`)
                  }
                }

Instanciation


                  let person1 = new Person('John Doe'); // create a person
                  person1.introducedMe(); // Hello, I am John Doe.

Decorator

Decorators are used to extend the behavior of a class, property, method, or method parameter without modifying the implementation.

Decorators are just functions

Note: the idea here is not to show how to create a decorator, but to give an idea on how it looks like.

Optional parameters

Example 1 : Class Decorator


                function nonEnumerable(target: any) {
                  ...
                }
  
                @nonEnumerable
                class Person {
                  ...
                }

Example 2 : Property decorator


                function ReadOnly(target: any, key: string) {
                  Object.defineProperty(target, key, { writable: false });
                }
                class Test {
                  @ReadOnly // notice there are no `()`
                  name: string;
                }
