REST

What is REST ?

  • A simple way to organize interactions between independent systems.
  • Not a protocol it is a way of doing.
  • Is Based on HTTP.

REST => Resource URI, HTTP Verbs, Representation.

Resource : Object associated with data identifdied with URI.

URI = Unique Resource Identifier.


              /users
              /users/{userId} # eg: /users/23
              /users/{userId}/comments
              /users/{userId}/comments/1234

HTTP Verbs : Used to indicate an action


                GET     -->   Show
                POST    -->   Create
                PUT     -->   Update
                DELETE  -->   Delete

Representation

  • XML
  • JSON

HTTP Response Status Code 


                # Success
                200 OK
                201 Created
                204 No Content

                # Redirection
                301 Moved Permanently
                302 Moved Temporary

                # User errors
                400 Bad Request
                401 Unauthorized
                404 Not Found
                403 Forbidden

                # Server errors
                500 Internal Server Error
                503 Service Unavailable
                504 Gateway Time-out

Examples

Create a Project 


            # -- Request --
            POST localhost:3000/api/projects
            Content-Type: application/json
            {
              "name": "project11",
              "startDate": "2018-09-19",
              "endDate": "2018-09-23",
              "description": "project Eleven",
            }

            # -- Response --
            201 Created
            {
              "targetSocialMedias": [],
              "_id": "5bd7118cc5bc6a25ce5ac478",
              "name": "project11",
              "startDate": "2018-09-19",
              "endDate": "2018-09-23",
              "description": "project Eleven",
              "__v": 0
            }
