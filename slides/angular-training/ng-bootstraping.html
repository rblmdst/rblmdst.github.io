Bootstrapping Angular Application

Purpose

Explain the bootstrap process of an Angular Application.

Angular Basic Building Blocks

The basic building blocks of an Angular application are modules.

Module = mechanism to group components, directives, pipes and services that are related.

Module are combined with other modules to create an application.

Bootstrapping

Bootstrapping is the process an angular application takes to come to life.

The application launches by bootstrapping the root module.

(By convention, the root module is usually called AppModule.)

The root module then create the component(s) listed in the bootstrap array and inserts each one into the browser DOM.

Root Module 


            import { BrowserModule } from '@angular/platform-browser';
            import { NgModule } from '@angular/core';
            import { FormsModule } from '@angular/forms';
            import { HttpClientModule } from '@angular/common/http';

            import { AppComponent } from './app.component';

            /* the AppModule class with the @NgModule decorator */
            @NgModule({
              declarations: [
                AppComponent
              ],
              imports: [
                BrowserModule, FormsModule, HttpClientModule
              ],
              providers: [],
              bootstrap: [AppComponent]
            })
            export class AppModule { }

NgModule

The basic NgModule


            ...
            // @NgModule decorator with its metadata
            @NgModule({
              declarations: [AppComponent, ItemDirective],
              imports: [BrowserModule,FormsModule,HttpClientModule],
              providers: [],
              bootstrap: [AppComponent]
            })
            export class AppModule { }

  • declarations: registers components within this module;
  • bootstrap: tells the module which component to bootstrap when bootstrapping the module;
  • imports: list of components to import (from other modules) into this module;
  • providers: list of services to inject in the module.
