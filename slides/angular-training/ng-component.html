Component

The core concept of any Angular application is the component.

Component is anything that is visible to the end user and which can be reused many times within an application.

Anatomy of Components

Component life-cycle

A component life-cycle, is different phases it goes through from birth (create) to death (destroy).

Component life-cycle

For more details visit: https://angular.io/guide/lifecycle-hooks

Hook onto a component life-cycle

Angular provides hooks to these life cycle events. These hooks can be used to detect when a certain life-cycle event occurs and help in performing certain actions during the course of the events.


              class AppComponent implements OnChanges, OnInit {
                constructor() {
                }
                ngOnChanges() {
                  console.log('ngOnChanges');
                }
                ngOnInit() {
                  console.log('ngOnInit')
                }
              }

Here is the complete lifecycle hook interface inventory:

  • ngOnChanges - called when an input binding value changes
  • ngOnInit - after the first ngOnChanges
  • ngDoCheck - after every run of change detection
  • ngAfterContentInit - after component content initialized
  • ngAfterContentChecked - after every check of component content
  • ngAfterViewInit - after component's view(s) are initialized
  • ngAfterViewChecked - after every check of a component's view(s)
  • ngOnDestroy - just before the component is destroyed

Global Styling

By default styles are limited to component in which they are defined.

When generating an Angular project using the angular-cli, a styles.css (less/sass) is created in the application /src folder.

Styles put in the styles.css (less/sass) are considered global so avalaible for all components.

Property binding

(Passing Data into a Component)

Property binding

Allows a component to be provided input data from the outside (its parent).

Property binding is created using @Input decorator.

The inputs attribute defines a set of parameters that can be passed down from the component's parent.

Property binding


              // hello.component.ts
              import {Component, Input} from '@angular/core';
              @Component({
                  selector: 'hello',
                  template: '
Hello, {{name}}
'
              })
              export class Hello {
                  @Input() name: string;
              }

Event binding

(Responding to Component Events)

Event binding

Allows a component to propagate data to the outside.

Event binding is created using @Output decorator.

Event binding


                // counter.component.ts
                import {Component, EventEmitter, Output} from '@angular/core';
                @Component({
                  selector: 'counter',
                  template: `
                    

                        
                    

                  `
                })
                export class Counter {
                  cpt: number = 0;
                  @Output() count: EventEmitter = new EventEmitter();
                  increment() {
                    this.cpt++;
                    this.count.emit(this.cpt);
                  }
                }

Using Two-Way Data Binding

Using Two-Way Data Binding

Two-way data binding combines the input and output binding into a single notation using the ngModel directive.

Using Two-Way Data Binding


              <input [(ngModel)]="name">

What this is doing behind the scenes is equivalent to :


              <input [ngModel]="name" (ngModelChange)="name=$event">

Create Component that support Two-Way Data Binding

Create Component that support Two-Way Data Binding

To create your own component that supports two-way binding, you must define an @Output property to match an @Input , but suffix it with the " Change".


              // counter.component.ts
              @Component({
                selector: 'counter',
                // ...
              })
              export default class Counter {
                  @Input() count: number = 0;
                  @Output() countChange: EventEmitter = new EventEmitter();
                  increment() {
                      this.count++;
                      this.countChange.emit(this.count);
                  }
              }

              // parent.component.ts
              @Component({
                  template:''
                  directives:[Counter]
              })
              class ParentComponent {
                  // ...
              }
